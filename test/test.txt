명령어 테스트
git restore --staged와 rm --cached 의 차이점은 뭘까
이렇게 내용을 변경하고서 다시 add를 하면 덮어쓰기
그러고 git restore --staged 를하면 변경없음. 그냥 워킹디렉토리로 내려옴
git rm --cached -r을 하니까 스테이징에 있던 파일이 삭제 후 워킹디렉토리로 내려옴..?
한번이라도 커밋을하면 파일은 추적(tracked)상태가 되는데,
restore는 추적상태를 유지하면서 스테이징->워킹디렉토리로 이동되나
rm --cached는 추적상태를 지우면서(untracked상태가됨) 스테이징->워킹디렉토리로 이동되는 차이가 있음
결론) 커밋전에는 차이가 없는 것처럼 보이나, 커밋 후에는 추적상태의 차이가 발생함!!
